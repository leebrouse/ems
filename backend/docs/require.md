# 救援物资需求申报与动态仓储/调度 — 需求文档 (Markdown)

> 目标：支持前端救援人员发起物资申报请求，管理人员审批并调度运输，运输过程可追踪、救援人员可查看物流并签收/拒收；管理端应明确展示「已送达」「已取消」等状态，并保证库存一致性、可追溯性与高可用性。

---

## 1. 参与角色（Actors）

* **救援人员（requester）** — 在前端发起物资申报（需求），查看物流与签收/拒收。
* **管理人员（dispatcher / approver）** — 审核申报、分配/调整物资、指派运输人员，查看总体状态（含已送达/已取消）。
* **运输人员（courier / driver）** — 接受并执行配送任务，上传运输状态（在途、到达、签收等）。
* **仓库操作员（warehouse operator）** — 负责入库/出库、核验物资与批次、处理异常。
* **系统（自动分配引擎）** — 根据库存、紧急程度、区域、有效期等给出分配建议（可选自动或半自动）。

---

## 2. 高层流程概览（Workflow）

1. 救援人员在前端填写并提交“物资申报单”。
2. 系统保存申报并（可选）进行初步校验与分配建议（基于库存/急迫性/距离/批次）。
3. 管理人员在管理界面审核申报：通过 / 驳回 / 需补充信息。
4. 若通过，管理人员生成配送任务并指派运输人员（或外包）。同时，系统在库存上做“预占/锁定”。
5. 运输人员接单并更新运输状态（已出库/运输中/已到达）。仓库操作员实际出库并更新批次、数量、剩余库存。
6. 救援人员可在其界面查看物流轨迹、预计到达时间、运输状态。到达后救援人员进行**签收**或**拒绝（退货/拒收）**。
7. 签收确认后系统将出库预占转为最终出库，库存最终扣减；若拒收，则进入退货/异常流程（可能触发库存回滚或二次处置）。
8. 管理界面应显示已送达 / 已取消 / 已拒收 / 已驳回 等关键状态并允许后续操作（例如退款、再派送、销毁过期批次）。

## 3. 功能需求（Functional Requirements）

### 3.1 申报（救援人员）

* 表单字段：

  * 申报单号（系统生成）
  * 申报人（自动填充）
  * 所属机构/站点
  * 物资列表（每行：物资ID、名称、规格、数量、优先级/紧急程度、预计使用日期、备注）
  * 目的地/接收地址与联系人（电话）
  * 期望交付时间窗口
  * 附件（照片、证明）
* 校验：

  * 必填字段检查、数量为正整数、联系人电话格式、单次申报上限规则
* 结果：

  * 成功提交后状态 `pending_review`，发送通知给管理人员

### 3.2 审批（管理人员）

* 列表视图：按 `pending_review` / `approved` / `rejected` 分页与筛选（按地区、优先级、时间段、物资类型）
* 详情页：

  * 申报单内容、历史日志、系统分配建议（含候选仓库与批次）、库存可用性
* 操作：

  * 批准（`approved`）并生成配送任务（可一起或分批）
  * 驳回（`rejected`）并填写原因（救援人员可见）
  * 请求补充信息（设置状态 `need_info`）
* 分配：

  * 手动分配或接受系统建议；分配时在库存上做“锁定/预占”（确保并发安全）
* UI 显示要求：

  * 必须直观显示“已送达（delivered）”或“已取消（cancelled）”等最终状态

### 3.3 配送与运输（运输人员）

* 运输任务表：任务ID、关联申报单号、指派人、接单状态、当前运输状态、预计到达时间
* 状态机（示例）：

  * `assigned` → `accepted` → `picked_up`（仓库已出库/取货） → `in_transit` → `arrived` → `delivered`（救援人员签收）
  * 额外：`rejected_by_recipient` → `return_pending` → `returned`
* 运输人员操作：

  * 接单、更新位置/状态、上传运输照片、签收证据（电子签名/照片）
* 救援人员操作：

  * 查看实时状态、查看预计ETA、签收（确认数量/完好）或拒收（并说明原因、可上传证据）

### 3.4 库存与出入库（仓库）

* 支持批次管理、有效期、单批次可用量、FIFO/FEFO 策略
* 出库操作需关联配送任务与申报单；出库确认后更新库存并记录批次与操作人
* 支持库存“预占/锁定”与“最终扣减”机制（支持事务/乐观锁或悲观锁）

### 3.5 异常与退货处理

* 拒收或配送异常需创建事件，包含原因、证据、建议处理方案（退回/补发/销毁）
* 管理人员可在界面处理异常并触发库存回滚或重新分配

### 3.6 审计与日志

* 每个关键动作（提交、审批、分配、出库、运输状态变更、签收/拒收）记录操作人、时间、前后状态、备注与证据
* 可导出日志（CSV/JSON）供审计使用

### 3.7 通知

* 事件驱动通知：提交/审批结果/派单/运输状态变更/签收/拒收/异常处理
* 通道：站内消息、邮件、短信（可选）、移动推送（可选）
* 模板与频率可配置（例如紧急申报即时通知）

---

## 4. 状态机（申报单与配送任务）

### 4.1 申报单（request）状态

* `draft` → `pending_review` → `approved` → `in_distribution` → `completed`（已送达）
* `pending_review` → `rejected`（已驳回）
* `approved` → `cancelled`（已取消）
* `completed` → `closed`（归档）

### 4.2 配送任务（shipment）状态

* `created` → `assigned` → `accepted` → `picked_up` → `in_transit` → `arrived` → `delivered`
* `created` / `assigned` → `cancelled`
* 在任何运输阶段都可能进入 `exception`（异常）→ `returned` / `resolved`

> 管理界面必须能够按以上状态高亮显示并提供筛选（尤其是 `delivered` 与 `cancelled`）

## 7. 用户界面（前端）需求要点

### 7.1 救援人员界面（移动/网页）

* 申报表单（一步或分步）；表单保存草稿
* 我的申报：列表 + 详情（显示审批状态、分配/物流信息、历史事件）
* 实时物流跟踪页（地图 + 时间轴 + ETA）
* 签收 / 拒收按钮（带拍照/电子签名上传）
* 通知中心（审批结果、派单、ETA 变化、异常）

### 7.2 管理人员界面（桌面优先）

* 仪表盘（待审批数量、待配送、运输中、异常、今日已送达、今日已取消）
* 审批队列（筛选、快速审批、批量审批）
* 申报详情页（含系统分配建议、库存批次、分配操作界面）
* 配送任务管理（指派、重派、取消、查看轨迹）
* 异常处理与退货管理
* 导出报表（按时间/物资/地区/人员）

### 7.3 运输人员界面（移动优先）

* 我的任务列表（接单/拒单、导航链接、上传事件/照片、到达确认）
* 状态更新快捷操作（一键拍照上传并标记 `picked_up` / `arrived` 等）
* 离线模式：记录事件并在恢复网络时同步

### UI 显示要求（关于已送达/已取消）

* 管理人员主视图应有显著过滤器/指示器展示 `delivered`（已送达）与 `cancelled`（已取消）项；状态卡片颜色或图标统一且有 legend；详情页显示最终签收信息或取消原因与时间戳。
