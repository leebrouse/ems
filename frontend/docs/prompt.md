你是一位精通 **Vue3 (TypeScript) + Vite + Element Plus + Pinia+ 高德地图api**（或者如果你用 React，请自行更改）的高级前端架构师。你擅长构建高性能、响应式的管理后台系统。

**项目背景：**
我正在开发一个“基于 Golang 的救援物资管理系统”的毕业设计。系统需要处理应急物资的流转、调拨和可视化监控。

**技术栈要求：**

* **UI 框架：** Element Plus (支持暗黑模式)
* **图表：** ECharts (用于物资统计)
* **地图：** 高德地图 API (用于物流轨迹)
* **状态管理：** Pinia
* **网络请求：** Axios (需包含 JWT 拦截器逻辑)

**待实现模块需求：**

1. **全局：** 请基于以下 API 文档，为我设计一套符合 **RBAC（基于角色的访问控制）** 的路由和菜单体系（包含救援人员、仓库管理员、系统管理员）。
2. **物资库存模块：** 实现一个带搜索、筛选、分页的表格。库存低于阈值时，行背景需显示红色预警。
3. **智能调度可视化：** 创建一个“需求申请”表单，并设计一个“调度大盘”，展示物资从仓库到灾区的流转状态。
4. **地图追踪：** 集成高德地图，根据后端返回的经纬度坐标串，绘制物资运输的动态轨迹。
5. **统计看板：** 使用 ECharts 生成物资消耗趋势折线图和类别占比饼图。

## 🛠️ 针对你 6 大模块的开发建议

为了让你的毕设更出彩，在生成代码时可以额外关注以下细节：

### 1. 权限与安全 (Auth)

* **细节：** 让 AI 生成一个 `useAuth` hook，处理 JWT 的存储（LocalStorage/Cookie）和过期自动跳转。
* **亮点：** 实现“按钮级”权限控制（例如：救援人员看不见“删除仓库”按钮）。

### 2. 库存预警 (Alerts)

* **可视化：** 别只用文字提示。
* **建议：** 在侧边栏或顶部导航栏设计一个“消息铃铛”，实时显示库存不足或物资过期的红点提醒。

### 3. 智能分配 (Smart Scheduling)

* **交互：** 这是一个流程性很强的功能。建议使用 **步骤条 (Steps)** 组件：
* `提交需求` -> `系统智能匹配` -> `管理员审核` -> `生成配送任务`。

### 4. 物流可视化 (Map)

* **技术点：** 告诉 AI ：“请帮我封装一个 `MapContainer.vue` 组件，支持传入 `path` 数组（经纬度列表），并在地图上自动缩放以适配完整路径。”

### 5. 性能保障 (Optimization)

* **前端方案：** 对于大批量物资列表，要求 AI 使用**虚拟滚动 (Virtual Scroll)**，这能体现你对系统性能的思考，是毕设的加分项。
